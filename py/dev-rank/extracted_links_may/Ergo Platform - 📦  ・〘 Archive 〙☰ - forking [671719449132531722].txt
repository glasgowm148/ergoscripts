@kushti the version 4.0 of Sigma will support both the current (Ergo Protocol v1) and the new (Ergo Protocol v2) versions at the same time depending on the parameters passed as part of context. Please see HFVoted, HFVotedBlock, HFBlock in EIP9 here https://github.com/ergoplatform/eips/blob/20e99a6f0c252a1f618bb9afa127ddefa3c4180f/eip-0009.md#phase-1-release-v400. 
Which version to use depends entirely on the activation protocol implemented on Ergo node side. So if you think it can be done as a SF later, that is a good news, I have only partial knowledge of how SF is implemented in Ergo node.
I'm currently working on complete executable specification which: 
1) cover the whole language 
2) test both v1 and v2 versions of Sigma
3) tests backward compatibility of v2 with v1
Once I finish with spec, I will resume implementation of missing parts of v2 version (for now they are postponed).
This is a lot of work ahead, so I will not be able to do anything on Ergo node side. 
But, keeping in mind that Sigma 4.0 will support both versions of Ergo Protocol (v1 and v2) the Ergo node activation protocol implementation can be done in parallel.
