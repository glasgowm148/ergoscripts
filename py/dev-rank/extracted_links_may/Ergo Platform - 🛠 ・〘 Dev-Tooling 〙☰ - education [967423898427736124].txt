[10/28/2022 11:59 AM] morphic#4133
I think the amount of information is overwhelming for newcomers. I suggest to structure the information around platforms/tech stacks the people are going to use: 
1) JVM - people can use Java or Scala or Kotlin, doesn't matter but they finally want to run on JVM
2) JS - people may use JavaScript or TypeScript or Scala.js, but they want to run on Node.js or in Browser
3) Android - people may use Java/Kotlin/Scala and want to run on Android
4) iOS - people may use Swift/Java+RoboVM/Objective-C and run on iPhone, iPad
5) Desktop - people may use Java+JVM/JavaFX/JS+Electron/Rust/Python
And then for each platform provide `recommended` starting points (1 or 2) and also list all related references, which are optional and not on the _getting started fast track_
For example: 
1) for JVM the starting point is Appkit and tutorial of createing dApp with Appkit
2) for JS - IDK, can be some SDK which is the most usable and have end-to-end example 
3) Android/iOS - is also Appkit and MrStahlfelge's tutorial
4) Desktop - I don't know, what is the top SDK here, but Appkit is also suitable and have working example (Desktop wallet)


[10/28/2022 12:00 PM] morphic#4133
So, the development tools depend on the platform, and this is the first thing anyone willing to develop need to ask themselves.
